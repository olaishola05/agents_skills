# Docs

Generate standard inline documentation for the provided code or file.

## Workflow

1. Read the target file or code snippet.
2. Analyze functions, classes, and complex logic.
3. Add appropriate documentation comments (e.g., JSDoc for JS/TS, docstrings for Python) focusing on parameters, return types, and exceptions.
4. Apply the changes to the file or output the documented code for review.

## Usage

```
/docs src/utils.ts
/docs --all
```
