# explain

Explain a file, function, or concept in plain English.

## Workflow

1. Take the target code, file, or architectural concept.
2. Break down its purpose, inputs, outputs, and side effects.
3. Explain the underlying logic step-by-step.
4. Provide context on how it fits into the broader system (if applicable).
5. Offer to answer follow-up questions.

## Usage

```
/explain src/complex-algo.ts
/explain "How does the auth flow work in this repo?"
```
